### Day 3-4：设计三阶段隔离工作流
**目标**：建立"可复现、可追溯、可控风险"的自动化内容生成流程

**核心原则**：三阶段隔离 + 渐进式披露 + 文件化产物

#### 阶段隔离架构
```
Stage 1：数据准备（脚本执行）
├─ 输入：知识库路径、选题库
├─ 处理：检索、抽取、验证
└─ 输出：evidence.md、facts.json、sources.md

Stage 2：内容创作（Agent决策）
├─ 输入：evidence.md、facts.json、文章模板
├─ 处理：大纲生成、内容写作、合规检查
└─ 输出：draft.md、citations.json、compliance_checklist.md

Stage 3：发布上线（脚本+人工确认）
├─ 输入：draft.md、人工审核标记
├─ 处理：格式转换、SEO优化、多渠道发布
└─ 输出：published.md、wechat.html、performance_tracking.json
```

**硬契约**：阶段2（内容创作）**只能读取** Stage 1 的输出文件（evidence.md/facts.json），不得直接读取原始知识库文件，避免上下文污染。

#### Stage 1：数据准备（脚本执行）
```yaml
输入文件：
- knowledge_base/: 知识库目录
- topics_pool.json: 选题库
- current_date: 当前日期

处理脚本：
1. 选题分析：分析知识库热点，检查时效性，平衡选题类型
2. 素材检索：从知识库检索相关案例、工具、政策
3. 证据抽取：提取关键数据点，去识别化处理
4. 来源标注：记录每个证据的来源、核验日期、可信度等级

输出文件：
- evidence.md: 核心证据块（结构化数据）

文件格式要求：

- 每个证据块有唯一ID

```

#### Stage 2：内容创作（Agent决策）
```yaml
输入文件：
- evidence.md: Stage 1 输出的证据块
- facts.json: Stage 1 输出的验证数据
- template/: 文章模板目录
- style_guide.md: 写作风格指南
- compliance_rules.yaml: 合规规则

处理Agent：
1. 大纲生成：根据evidence.md和模板生成详细大纲
2. 内容创作：基于大纲和facts.json写作
3. 合规检查：检查compliance_rules.yaml所有规则
4. 来源标注：在正文中标注每个关键判断的来源ID

输出文件：
- draft.md: 文章草稿
- citations.json: 引用标注
- compliance_checklist.md: 合规检查清单
- quality_report.md: 质量检查报告

硬约束：
- Agent只能读取evidence.md和facts.json，不能访问原始知识库
- 每段关键判断必须有citations.json中的对应引用
- 必须通过所有compliance_rules.yaml的检查

class TitleABGenerator:
    def generate_title_options(self, topic):
        """生成多个标题选项用于AB测试"""
        titles = []
        # 痛点共鸣型（不涉承诺）
        titles.append(f"为什么你的{topic}总是达不到银行要求？")
        # 过程导向型
        titles.append(f"解决{topic}问题的3个可执行步骤")
        # 价值承诺型
        titles.append(f"{topic}改善方案：具体任务清单与证据要求")
        # 数字清单型
        titles.append(f"提升{topic}效率的5个关键动作")
        return titles
    
    def compliance_check(self, title):
        """检查标题合规性"""
        forbidden_phrases = ["保证", "包过", "必下款", "100%通过", "秒批"]
        for phrase in forbidden_phrases:
            if phrase in title:
                return False, f"包含禁止表述: {phrase}"
        return True, "合规"
```
