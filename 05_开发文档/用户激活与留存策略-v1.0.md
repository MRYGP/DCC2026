**版本**：v1.0  
**创建日期**：2026-02-09  
**适用范围**：Phase 0-1（V0验证期 + V1扩展期）  
**依赖文档**：产品需求文档-V0版-v1.4.md  
**责任人**：产品团队 + 运营团队  
**状态**：已发布

## 文档说明

本文档定义贷款中介业务基座的用户激活与留存策略，基于Hook模型（上瘾模型）设计完整的用户习惯养成机制。

**核心目标**：
- 让中介从"偶尔用用"变成"每天主动打开"
- 活跃天数从3天/周 → 5天/周
- 30日留存率 > 40%

**适用对象**：
- V0阶段：5个朋友验证
- V1阶段：10-30个中介扩展

---

## 一、战略目标

### 1.1 北极星指标

**核心指标**：活跃天数（DAU/MAU）
- **V0目标**：从3天/周 → 5天/周
- **V1目标**：从5天/周 → 6天/周

### 1.2 关键行为指标

| 指标 | V0目标 | V1目标 | 说明 |
|------|--------|--------|------|
| **诊断次数** | 5次/周 | 10次/周 | 核心价值使用频率 |
| **导入客户数** | 50个/朋友 | 100个/中介 | 数据资产积累 |
| **方案生成次数** | 2次/周 | 5次/周 | 深度功能使用 |
| **结果回填率** | >50% | >80% | 数据飞轮关键 |

### 1.3 留存指标

| 留存周期 | V0目标 | V1目标 |
|---------|--------|--------|
| **次日留存** | >70% | >80% |
| **7日留存** | >50% | >60% |
| **30日留存** | >40% | >50% |

## 二、Hook模型应用

### 2.1 Hook模型四要素

```
触发（Trigger）→ 行动（Action）→ 奖励（Reward）→ 投入（Investment）
     ↑                                                        ↓
     ←─────────────────── 循环强化 ──────────────────────────┘
```

**核心逻辑**：
1. **触发**：让用户想起要打开系统
2. **行动**：降低行动门槛，让操作更简单
3. **奖励**：给予即时反馈，让用户有成就感
4. **投入**：让用户投入数据/时间，增加退出成本

### 2.2 触发设计（Trigger）

#### 外部触发

**触发1：银行政策更新提醒** ⭐⭐⭐⭐⭐
- **频率**：每周2-3次（周一、周三、周五）
- **渠道**：微信服务号推送 + 系统内红点
- **文案模板**：
```
【政策更新】浦发银行-浦慧税贷
准入条件放宽：年开票从300万降至200万
影响您的客户：8个B类客户可升为A类
→ 点击查看受影响客户清单
```
- **关键**：个性化（"影响您的8个客户"）+ 可执行（点击直接看清单）

**触发2：客户状态变化提醒** ⭐⭐⭐⭐⭐
- **频率**：每天1次（早上9:00）
- **场景**：
  - A类客户到期提醒："客户张三的流水已满6个月，现在可申请XX银行产品"
  - B类客户改造完成提醒："客户李四已完成开票提升，从B类升为A类"
  - C类客户复评提醒："客户王五距离上次诊断已6个月，建议重新诊断"
- **文案模板**：
```
【客户提醒】您有2个客户可立即跟进
• 张三 - 流水已满足浦发银行条件，预计可贷50万
• 李四 - 高新认定已完成，B类→A类
→ 点击查看详情并联系客户
```

**触发3：竞争压力提醒（社交证明）** ⭐⭐⭐
- **频率**：每周1次（周五下午）
- **场景**：
  - 同行成交案例："本周已有3个中介通过系统成交，您的客户XX也符合条件"
  - 行业动态："本月小微企业贷款审批率提升12%，您的B类客户可重新尝试"

#### 内部触发

**触发4：习惯培养**
- **目标**：让用户形成"每天早上9点打开系统"的习惯
- **方法**：
  - 固定时间推送（早上9:00）
  - 固定行为奖励（打开系统 → 看到待办清单 → 完成1个任务 → 获得成就感）
  - 21天习惯养成计划（V1功能）

### 2.3 行动设计（Action）

#### 降低行动门槛

**原则**：从3步变1步，从1分钟变10秒

**场景1：政策更新后的行动路径**
```
传统流程（5步，2分钟）：
1. 看到政策更新
2. 登录系统
3. 手动筛选客户
4. 逐个查看
5. 复制手机号联系

优化后（2步，10秒）：
1. 点击提醒 → 直接看到"受影响的8个客户清单"（已筛选好）
2. 点击客户 → 一键发送短信模板（跳转微信，预填文案）
```

**场景2：客户提醒后的行动路径**
```
优化前（5步）：
1. 看到提醒
2. 查客户详情
3. 复制手机号
4. 切到微信
5. 发消息

优化后（2步）：
1. 点击提醒
2. 点击"发送微信" → 跳转微信（预填文案）→ 发送
```

#### 首页待办清单设计

**设计原则**：首页就是行动清单，不是数据看板

**首页结构**：
```
┌─────────────────────────────────────┐
│ 今日待办                              │
├─────────────────────────────────────┤
│ 🔴 3个A类客户可立即跟进               │
│    → 查看详情                         │
│ 🟡 5个B类客户改造到期                 │
│    → 查看任务清单                     │
│ 🟢 2个新客户等待诊断                  │
│    → 开始诊断                         │
├─────────────────────────────────────┤
│ 本周政策更新                          │
│ • 浦发银行-条件放宽（影响8个客户）      │
│ • 民生银行-利率下调（影响12个客户）     │
└─────────────────────────────────────┘
```

**关键**：
- 每个待办都可以一键操作
- 数字具体化（"3个A类客户"比"有客户待跟进"有效3倍）
- 颜色编码（红黄绿）降低认知负担

### 2.4 奖励设计（Reward）

#### 即时奖励（操作后立即反馈）

**场景1：导入客户后**
```
✅ 成功导入50个客户

发现：
• 15个A类客户（可立即跟进）
• 20个B类客户（值得改造）
• 15个C类客户（暂不跟进）

推荐行动：
→ 查看15个A类客户详情
→ 为20个B类客户生成改造方案
```

**场景2：完成诊断后**
```
✅ 诊断完成

好消息：客户张三从B类升为A类！

可申请产品：
• 浦发银行-浦慧税贷（最高50万）
• 民生银行-民生税贷（最高30万）

→ 立即联系客户
```

**场景3：方案生成后**
```
✅ 改造方案已生成

任务清单：
• 任务1：开票提升（30天完成）
• 任务2：纳税提升（60天完成）

预计效果：
改造后可贷额度从30万提升至80万

→ 查看完整方案
```

#### 社交奖励（排行榜/成就）

**每周排行榜**（朋友验证阶段可见）：
```
本周成交榜：
🥇 朋友A：5单成交
🥈 朋友B：3单成交
🥉 朋友C：2单成交
   您：1单成交（再成交1单即可进入前三！）
```

**成就系统**（V1功能）：
- 🎖️ 初次成交：完成第1单
- 🏆 客户管理大师：管理100+客户
- 💎 改造专家：帮助10个B类客户升为A类
- ⭐ 系统达人：连续使用30天
- 🔥 数据贡献者：结果回填率100%

#### 猎奇奖励（不可预测）

**随机推荐**（每天变化）：
```
💡 今日推荐

系统发现：您的客户李四可能符合"科小贷"条件

原因：
• 年开票300万+纳税15万
• 科技行业
• 无征信风险

推荐理由：
利率比普通产品低0.5%，可节省利息2500元/年

→ 查看详情
```

**关键**：不要每天推荐同样的东西，要有惊喜感

### 2.5 投入设计（Investment）

#### 数据投入（用得越多越离不开）

**积累数据**：
- 客户数据（导入越多，价值越大）
- 诊断历史（可追踪客户变化）
- 改造记录（成功案例积累）
- 成交记录（业绩可视化）

**数据价值展示**：
```
┌─────────────────────┐
│ 您的数据资产         │
├─────────────────────┤
│ 管理客户：150个      │
│ 诊断次数：320次      │
│ 成交记录：12单       │
│ 数据完整度：85%      │
├─────────────────────┤
│ 提示：数据完整度越高，│
│ 系统推荐越准确        │
│ → 完善客户资料        │
│   （还差15%）         │
└─────────────────────┘
```

#### 个性化投入（定制化配置）

**让用户配置系统**（V1功能）：
- 自定义提醒时间（"我习惯早上9点看客户提醒"）
- 自定义标签（"我把客户分为：意向强/观望/冷客户"）
- 自定义筛选条件（"我只看年开票>200万的客户"）

**关键**：配置越多，迁移成本越高

#### 社交投入（邀请好友）

**V1功能**（V0不做）：
- 邀请同行使用（"邀请3个朋友，解锁高级功能"）
- 案例分享（"分享成功案例到朋友圈，获得曝光"）

## 四、策略优先级矩阵

## 七、反面教材（避免踩坑）

### 7.1 不要做的事 ❌

#### 1. 不要过度推送
- **错误**：每天10条提醒 → 用户直接关闭通知
- **正确**：每天1-2条，集中在早上9点

#### 2. 不要推送无关信息
- **错误**：
  - "系统更新公告" → 用户不关心
  - "新功能上线" → 用户不关心
- **正确**：只推送和用户客户相关的内容

#### 3. 不要泛泛提醒
- **错误**："您有客户可以跟进"
- **正确**："客户张三流水已满足条件，可贷50万"

#### 4. 不要只有触发没有奖励
- **错误**：推送"政策更新" → 用户打开后自己去找
- **正确**：推送"政策更新" → 直接展示受影响的8个客户
