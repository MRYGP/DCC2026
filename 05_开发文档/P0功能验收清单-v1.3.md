## 0. 当前战场（朋友验证版）

**阶段**：Phase 0 - 朋友验证（13周：2周开发 + 11周验证）

**本轮焦点**：验证朋友是否真用、是否好用、是否有价值

**V0闭环**：批量导入 → 诊断分层 → 产品匹配 → 方案生成 → 提醒跟进

**验收目标**：
- 10项核心功能全部可验收
- 3-5个朋友完成实际测试并反馈"真用"
- 端到端流程可跑通
- 朋友使用数据达标（活跃天数>3天/周，诊断次数>5次/周）

## 0.1 V0部署形态裁决（重要调整）

**V0采用：独立部署（每个朋友独立实例）**

**交付物**：
- 独立部署包（Docker容器）
- 一键部署脚本
- 每个朋友独立数据库
- 独立域名访问（friendX.yourdomain.com）

**理由**：
1. 朋友更信任数据在自己手里
2. 完全隔离，一个朋友问题不影响其他
3. 品牌独立，每个朋友有自己的体验
4. 成本可控（5个容器，每月250元）

**影响的验收口径**：
- 功能9（白标配置）：验证Logo/名称配置生效
- 数据隔离：验证朋友间数据不可见
- 一键部署：验证部署脚本正常工作

**V1规划**：
- 根据验证结果决定：继续独立部署 or 转向多租户SaaS

## 1. 全局验收总则（所有功能共用）

### 1.1 角色与权限（V0简化版）

- 仅支持"单一管理员"角色（朋友本人）
- 完整系统权限，无分级权限控制
- V1再考虑多角色权限体系

### 1.2 核心业务规则（必须一致）

**分层阈值**：
- A类（立即可贷）：总分 ≥ 600
- B类（值得改造）：400 ≤ 总分 < 600
- C类（暂不跟进）：总分 < 400

**六宫格评分权重**：
- 流水体质：200分
- 票税体质：200分
- 征信体质：200分
- 资产体质：150分
- 资质体质：150分
- 行业体质：100分
- **总分**：1000分

**算法护栏**（强制）：
- 可解释：每个分层能说明基于哪些字段/规则
- 人工复核：允许A↔B改判且留痕
- 免责声明：所有报告必带且不可删除

**一票否决项**：
- 法人征信有M3+逾期记录（连续3个月以上逾期）
- 法人征信有代偿记录（如果字段存在）

### 1.3 数据验证规则（朋友验证版）

**验证数据**：
- 每个朋友导入50+真实客户数据
- 数据包含13个必填字段
- 数据真实有效（非测试数据）

**验证方式**：
- 不看朋友口头评价，看使用数据
- 不看功能完整性，看核心流程跑通
- 不看一次使用，看持续使用

## 2. P0功能验收清单（10项核心功能）

### 功能1：批量导入Excel（P0）

**输入**：
- Excel文件（.xlsx格式）
- 包含13个必填字段

**13个必填字段**：
1. 企业名称（文本）
2. 注册资本（数值，单位万元）
3. 成立时间（日期，格式YYYY-MM-DD）
4. 法人姓名（文本）
5. 年开票（数值，单位万元）
6. 年纳税（数值，单位万元）
7. 对公流水月均（数值，单位万元）
8. 法人征信（文本，如"无逾期"/"M1逾期"/"M3+逾期"）
9. 企业征信（文本，如"正常"/"异常"）
10. 主营业务（文本）
11. 行业（文本，如"制造业"/"服务业"）
12. 贷款需求（数值，单位万元）
13. 抵押物情况（文本，如"有房产/150万"/"无"）

**处理逻辑**：
1. 文件格式验证（必须是.xlsx）
2. 字段完整性验证（13个字段必须存在）
3. 数据类型验证（数值字段必须是数字，日期字段必须是日期）
4. 数据清洗（去除空行、处理异常值）
5. 批量插入数据库

**输出**：
- 成功导入数量
- 失败记录列表（含错误原因）
- 导入统计报告

**验收标准**：
- [ ] 能上传.xlsx文件
- [ ] 能解析13个必填字段
- [ ] 能验证数据类型（数值、日期）
- [ ] 能批量插入数据库
- [ ] 100条数据5秒内完成
- [ ] 错误信息清晰可理解
- [ ] 朋友能独立完成导入操作

**简化点**：
- 只做必填字段验证，不做复杂格式校验
- 错误提示简单明了
- 支持100条数据5秒内完成

### 功能2：自动诊断分层（P0）

**输入**：
- 客户13字段数据
- 六宫格评分规则

**处理逻辑**：
1. 计算六宫格各项得分
2. 汇总总分
3. 根据阈值分层（A/B/C）
4. 生成分层理由

**六宫格评分规则**：
1. 流水体质（200分）
   - 对公流水月均：100分
   - 流水稳定性：100分
2. 票税体质（200分）
   - 年开票额：100分
   - 年纳税额：100分
3. 征信体质（200分）
   - 法人征信：200分
4. 资产体质（150分）
   - 抵押物价值：150分
5. 资质体质（150分）
   - 资质认定：150分
6. 行业体质（100分）
   - 行业风险：100分

**输出**：
- 分层结果（A/B/C）
- 总分（0-1000）
- 各项得分明细
- 分层理由（基于哪些字段）

**验收标准**：
- [ ] 能计算六宫格各项得分
- [ ] 能汇总总分
- [ ] 能根据阈值正确分层（A≥600, 400≤B<600, C<400）
- [ ] 能生成分层理由
- [ ] 能处理一票否决项（M3+逾期→C类）
- [ ] 单个客户诊断≤3秒
- [ ] 朋友能理解分层结果

### 功能3：产品匹配（P0）

**输入**：
- 客户分层结果（A/B/C）
- 客户详细信息
- 银行产品库（20个产品）

**处理逻辑**：
1. A类客户：匹配立即可贷产品
2. B类客户：匹配改造后可贷产品
3. C类客户：不匹配产品，建议暂不跟进
4. 识别具体差距（数值型差距量化）

**匹配规则**：
1. 遍历产品准入条件
2. 逐条对比客户条件
3. 筛选符合条件的产品
4. 按匹配度排序
5. 输出差距清单

**输出**：
- 匹配产品列表（最多5个）
- 推荐理由
- 差距清单（如：流水不足XX万）
- 预估额度范围

**验收标准**：
- [ ] A类客户能匹配到产品
- [ ] B类客户能识别差距
- [ ] C类客户建议暂不跟进
- [ ] 匹配理由清晰可理解
- [ ] 差距清单具体可量化
- [ ] 匹配计算≤2秒
- [ ] 朋友能理解匹配结果

**简化点**：
- 先匹配5个最常用产品（覆盖80%场景）
- 详细差距分析可简化
- 推荐理由简单明了

### 功能4：方案生成（P0）

**输入**：
- 客户分层结果（B类）
- 具体差距清单
- 改造方案库（12个方案）

**处理逻辑**：
1. 根据差距类型匹配改造方案
2. 生成结构化方案包
3. 计算改造成本和ROI
4. 推荐服务商（可选）

**方案包结构**：
1. 缺口清单（差什么、差多少）
2. 任务清单（做什么、谁负责、证据要求、完成标准）
3. 推荐服务商（2-3家可选）
4. 效果预览（改造后预计能贷多少）
5. 成本估算（改造成本、ROI）

**输出**：
- 结构化方案包（可导出PDF）
- 任务清单（可执行）
- 服务商推荐（可选）
- 成本效益分析

**验收标准**：
- [ ] B类客户能生成改造方案
- [ ] 方案包含5个标准部分
- [ ] 任务清单具体可执行
- [ ] 效果预览合理可信
- [ ] 成本估算有依据
- [ ] 方案生成≤5秒
- [ ] 朋友能理解并执行方案

**简化点**：
- 从12个方案库匹配简化为3个核心方案
- ROI计算可简化
- 服务商推荐可简化

### 功能6：可解释+人工复核（P1）

**人工复核流程**：
1. 查看系统分层结果
2. 查看分层理由
3. 决定是否改判
4. 输入改判理由
5. 系统记录改判记录

**输出**：
- 分层理由详情
- 人工复核记录
- 算法优化数据

**验收标准**：
- [ ] 能展示分层理由详情
- [ ] 支持人工改判A↔B
- [ ] 记录改判理由
- [ ] 改判记录可查询

## 7. 验收通过标准

### 7.1 功能验收通过标准（全部满足）
- ✅ 10项核心功能全部可用
- ✅ 核心流程可走通（导入→诊断→方案→提醒）
- ✅ 无致命bug
- ✅ 朋友能独立使用

### 7.2 朋友验收通过标准（全部满足）
- ✅ 5个朋友都在用（活跃率>80%）
- ✅ 至少2个朋友明确说"很有用"
- ✅ 至少1个朋友因系统成交1单
- ✅ 建立了快速响应机制（1周内修复问题）

### 7.3 数据验收通过标准（全部满足）
- ✅ 导入客户数 > 50个/朋友
- ✅ 活跃天数 > 3天/周
- ✅ 诊断次数 > 5次/周
- ✅ 方案生成次数 > 2次/周

### 7.4 验收决策树

```
第13周验收 → 评估结果 → 决策下一步

分支1：全部达标（理想情况）
├── 继续Phase 1：扩展到30个中介
├── 开始收费准备：1-2万/年象征性收费
├── 产品化优化：基于真实反馈优化
└── 扩大测试范围：更多朋友加入

分支2：部分达标（常见情况）
├── 分析原因：哪个标准没达标？
├── 针对性优化：针对薄弱环节改进
├── 延长测试周期：再测试1个月
└── 重新评估：调整成功标准

分支3：全部不达标（最差情况）
├── 深度访谈：找真实原因
├── 产品方向调整：可能做错了
├── 重新定义MVP：更小的功能集
└── 考虑放弃或转型
```
