# 贡献指南 / Contributing Guide

感谢你对贷款中介业务基座项目的关注！

---

## 🤝 如何贡献

### 知识库贡献

**银行产品更新**：
- 文件位置：`01_银行政策库/`
- 格式：参考现有产品文档
- 必填：额度、利率、准入条件、额度算法

**改造方案补充**：
- 文件位置：`02_改造方案库/`
- 格式：参考现有方案文档
- 必填：适用场景、核心步骤、周期、成本、ROI

**案例分享**：
- 文件位置：`04_案例库/客户案例/`
- 格式：参考`案例00-模板.md`
- 必填：客户背景、诊断过程、改造方案、效果、教训

### 技术贡献

**代码提交**：
1. Fork本仓库
2. 创建特性分支：`git checkout -b feature/功能名`
3. 提交代码：`git commit -m "feat: 简短描述"`
4. 推送分支：`git push origin feature/功能名`
5. 创建Pull Request

**Bug报告**：
- 使用Issue模板
- 描述清楚：复现步骤、预期结果、实际结果
- 附上截图或日志（如有）

## 📝 文档规范

### 命名规范

**文件命名**：
- 中文：`XX银行-XX贷.md`
- 英文：`v0-development-plan.md`
- 不要用空格，用短横线

**标题规范**：
```markdown
# 一级标题（文档标题）

## 二级标题（章节）

### 三级标题（小节）
```

### 内容规范

**原则**：
1. 简洁：能用300字说清的，不要3000字
2. 实用：必须能指导实际操作
3. 准确：数据必须可验证

**禁止**：
- ❌ 大段铺垫（"众所周知"等）
- ❌ 过度格式化（大量emoji、加粗）
- ❌ 重复内容
- ❌ 无法验证的数据

---

## ✅ 提交规范

### Commit信息格式

```
<type>: <subject>

<body>（可选）
```

**Type类型**：
- `feat`: 新功能
- `fix`: Bug修复
- `docs`: 文档更新
- `refactor`: 代码重构
- `test`: 测试相关
- `chore`: 构建/工具变动

**示例**：
```
feat: 新增农业银行纳税e贷产品文档

- 补充额度算法
- 添加准入条件
- 完善材料清单
```
