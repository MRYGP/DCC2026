## 一、框架总览

### 1.1 研究目的

**为什么要做竞品深度研究？**
- 避免闭门造车，借鉴成功经验
- 发现竞品的盲区和空白点
- 找到差异化竞争的切入点
- 预判行业风险和机会

**我们想从中得到什么？**
1. ✅ **可借鉴清单**：直接可以抄的产品功能、运营策略、话术
2. ⭐ **差异化机会**：竞品不做/做不好/不会做的地方
3. ❌ **打不过清单**：明确哪些领域不要硬刚
4. 🎯 **行动建议**：基于分析的具体执行计划

## 二、第一部分：基本面扫描（事实层）

### 模块1：公司基本面

#### 1.1 必填字段

| 字段 | 说明 | 数据来源 |
|------|------|---------|
| 公司名称 | 全称 + 简称 | 官网 |
| 成立时间 | YYYY-MM | 工商信息 |
| 总部地点 | 城市/国家 | 官网 |
| 当前状态 | 运营中/被收购/倒闭/转型 | 新闻 |
| 融资历史 | 每轮金额/投资人/时间 | Crunchbase/IT桔子 |
| 当前估值 | 最后一轮估值 | 融资新闻 |
| 员工规模 | 人数 | LinkedIn/天眼查 |
| 核心指标 | 用户数/GMV/放款量/收入 | 公开报道/推测 |
| 关键时间线 | 重要里程碑 | 历史新闻 |

#### 1.2 分析要点

**融资节奏分析**：
- 融资间隔说明什么？（快=高速增长，慢=谨慎/困难）
- 投资人背景说明什么？（战略资源/财务投资）
- 估值变化说明什么？（增长速度/市场预期）

**规模增长曲线**：
- 用户增长是否健康？（指数增长/线性增长/停滞）
- GMV/放款量增长是否匹配用户增长？
- 有没有关键转折点？

**当前状态判断**：
- 如果被收购：为什么被收购？（成功退出 vs 被迫卖身）
- 如果倒闭：什么原因？（资金链 vs 模式问题）
- 如果转型：为什么转？转向哪里？

#### 8.4 综合评分

```
总分 = 天时×30% + 地利×30% + 人和×40% = ____/10
```

**评分解读**：
- **8-10分**：三者兼备，成功概率高

### 模块9：成败归因分析

#### 9.1 官方叙事

**他们声称的成功原因是什么？**
- 来源：官网/PR稿/创始人采访
- 记录原话，不要解读

#### 9.2 归因陷阱检查

| 陷阱 | 检查结果 | 说明 |
|------|---------|------|
| **因果倒置** | ✅/❌ | "因为成功所以XX" vs "因为XX所以成功" |
| **可见当关键** | ✅/❌ | 看得见的不一定是关键因素 |
| **故事当公式** | ✅/❌ | 成功故事不等于可复制公式 |

**避免被忽悠的3个问题**：
1. 如果没有这个因素，还能成功吗？
2. 别的公司也有这个因素，为什么没成功？
3. 这个因素是原因还是结果？

#### 9.3 真实成功因素（Top 3）

**排序原则**：
- 第一位：没有它就死定了
- 第二位：有它能加速成功
- 第三位：锦上添花

#### 9.4 关键转折点

**识别标准**：
- 什么时候/什么事情**改变了命运**？
- 如果没有这个转折点会怎样？

#### 9.6 可复制性分析

**三个拷问**：
1. **他们的成功可以复制吗？**
   - 需要什么条件？
   - 哪些条件我们有/没有？

## 九、常见问题

**Q1：必须填完所有12个模块吗？**
A：不必须。根据目的选择重点模块：
- 快速了解：模块1/2/11（基本面+对标）
- 深度学习：全部模块
- 竞争应对：模块5/7/12（定位+壁垒+风险）

**Q2：数据不完整怎么办？**
A：
1. 标注"推测"和"确认"
2. 给出推测依据
3. 列入"待验证"清单

**Q3：和行长/专家怎么聊？**
A：
- 不要问"你觉得这个竞品怎么样"（太大）
- 要问具体问题："如果开票提升50%，额度能增加多少？"（可回答）

**Q4：多久更新一次？**
A：
- 重点竞品：每季度更新
- 一般竞品：每半年更新
- 重大事件时：立即更新
