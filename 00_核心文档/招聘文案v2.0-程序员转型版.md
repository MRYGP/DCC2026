## 你会经历什么

### Week 1-2：理论武装（打基础）

**输入**

- 创始人的 42 篇认知文档（精选 10 篇必读）
- 7 步拆解法 + AI 项目评估系统
- DCC2026 产品逻辑 + 技术架构

**输出**

- 产品理解文档（能给别人讲清楚"贷查查是干什么的"）
- 竞品分析报告（找到 3 个差异化点）
- 第一个产品 Skill（手写 YAML 配置）

**验收**

- 能说出"为什么用 Skill 系统"（产品原因）
- 能提出至少 1 个优化建议
- 展现出"产品思维"

**不通过怎么办**：转为纯开发实习生，或友好结束

**时间投入建议**

- **全职实习生**
  - Week 1-2：每周 40 小时（8 小时/天 × 5 天）
  - Week 3-8：每周 40-50 小时（高峰期可能需要加班）
- **非全职实习生**
  - Week 1-2：每周 20-25 小时（5 小时/天 × 5 天）
  - Week 3-8：每周 25-30 小时
  - 建议：至少保证每天有连续 3-4 小时的专注时间
- **远程实习生**
  - 需要严格的时间管理（推荐用番茄工作法）
  - 每周至少 1 次视频 Review（30 分钟-1 小时）
  - 重要节点需要到现场（Week 2 验收 / Week 8 答辩）

---

### Week 3-6：工程交付（做产品）

**输入**

- FastAPI + React + Ant Design
- Skill 匹配引擎设计文档
- 质量闸门标准（pytest + ruff）

**输出**

- 后端：Skill 匹配引擎 + 热更新 API
- 前端：客户导入表单 + 结果展示页面
- 测试：60%+ 覆盖率 + 单位陷阱测试

**验收**

- 功能正常运行
- 能说清楚设计思路
- 能演示完整流程

---

### Week 7-8：复盘迭代（可上线）

**输入**

- 集成测试清单
- Docker 部署文档
- 复盘模板

**输出**

- 可演示的 V0 系统
- 部署到测试环境
- 成长复盘文档 + 给下一个实习生的建议

**验收**

- 能对外演示（投资人/合作伙伴）
- 技术 + 产品 + 协作三项达标
- 写出可复用的经验文档

## 面试流程

### Step 1：简历初筛（1天）

```
看什么：
- 有没有Web开发经验（Python + React）
- 有没有完整项目（课程作业/个人项目/实习项目）
- 有没有用过AI工具（Cursor/Copilot/Claude）

通过率：30%
```

### Step 2：技术测试（2小时）

```
测什么：
- 实现3个API接口（preview/match/report）
- 写测试用例验证功能
- 用AI辅助开发（Cursor/Claude Code）

通过率：50%
```

### Step 3：产品思维面试（30分钟）

```
问什么：
- 你用过什么AI产品？为什么喜欢/不喜欢？
- 如果让你改进这个产品，你会怎么做？
- 你为什么想做AI产品工程师？

通过标准：
✅ 能说出"为什么"（不是背答案）
✅ 有自己的思考
✅ 展现出好奇心

通过率：70%
```

### Step 4：最终决定（1周内）

```
综合评估：
- 技术能力（40%）
- 产品思维（30%）
- 学习能力（20%）
- 沟通能力（10%）

录取率：最终10-20%
```

## 常见问题FAQ

### Q1：我没用过FastAPI，能参加吗？

```
A：能。
- 只要有Python + Web框架经验（Flask/Django）
- Week 1会有FastAPI快速上手
- 重点不是技术栈，而是产品思维
```

### Q2：我只会后端/前端，不会全栈，能参加吗？

```
A：能，但有前提。
- 如果你会后端：Week 5-6会教前端基础
- 如果你会前端：Week 3-4会教后端基础
- 但要愿意学习"不熟悉的部分"
```

### Q3：2个月后一定能转正吗？

```
A：不一定。
- 需要通过Week 2验收（产品思维）
- 需要通过Week 8验收（技术+产品+协作）
- 如果不通过，会给出明确原因和建议
```

### Q4：如果我只想学理论，不想写代码，可以吗？

```
A：不可以。
- 这是"产品工程师"培养计划，不是"研究员"
- 必须有代码交付
- 如果只想学理论，建议读三向问道文档
```

### Q5：我能远程实习吗？

```
A：不可以，必须长沙线下。

为什么不支持远程：
- AI时代的产品工程师培养，需要高频面对面沟通
- Week 1-2的"产品思维训练"，远程效果差很多
- 代码Review、三拷问、验收会议，都需要现场

特殊情况：
- 家里有事：可以临时请假1-2天
- 回家过节：提前协调，不影响验收节点
```
