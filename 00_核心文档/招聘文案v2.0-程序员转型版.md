## 这个岗位的核心工作是什么

### 一句话版本

把银行的贷款产品翻译成机器能读的配置文件（我们叫Skill），然后写程序让这些配置跑起来，自动判断"这个客户能不能贷、能贷多少、差在哪里"。

### 展开说

银行有几十个贷款产品，每个产品有不同的准入条件（流水要多少、纳税要多少、征信不能太差）。
中介现在靠经验和记忆去匹配客户和产品——效率低、容易出错。

我们要做的是：
1. 把每个银行产品的规则写成一个Skill配置文件（YAML格式）
2. 写一段程序，拿客户数据去逐个比对所有Skill
3. 输出一份报告：哪些产品能贷、预估多少额度、不符合的差在哪里

**你的主要工作 = 设计Skill + 验证输出**。写代码是让Skill跑起来的手段，不是目的。

### 具体例子

假设工商银行有一个"税贷"产品，准入条件是：月流水≥10万、年纳税≥1万、近6月征信查询≤6次。

传统做法（硬编码）：
```python
def check_icbc_tax_loan(customer):
    if customer.monthly_revenue < 100000:
        return False, "月流水不足10万"
    if customer.annual_tax < 10000:
        return False, "年纳税不足1万"
    if customer.credit_query_count > 6:
        return False, "征信查询超过6次"
    return True, f"预估额度{customer.annual_tax * 25}元"
```

我们的做法（Skill 配置，符合《Skill格式规范》最小可运行结构）：

```yaml
# 元数据为唯一权威源（必填）
metadata:
  skill_id: "PRODUCT_ICBC_TAX_001"
  skill_name: "工商银行税贷"
  skill_type: "product"
  source: "工商银行官网"
  verified_date: "2026-02-01"
  verified_by: "产品"
  region: "全国"
  version: "1.0.0"
  created_by: "admin@dcc2026.com"
  status: "active"

eligibility:
  logic: "all"
  conditions:
    - field: "monthly_revenue"
      operator: ">="
      value: 100000
      description: "月均流水≥10万元"
    - field: "annual_tax"
      operator: ">="
      value: 10000
      description: "年纳税≥1万元"
    - field: "credit_query_6m"
      operator: "<="
      value: 6
      description: "近6月征信查询≤6次"

amount_calculation:
  formula_type: "expression"
  formula: "annual_tax * 25"
  amount_range: { min: 0, max: 5000000, unit: "元" }
```

**区别在哪**：

- 硬编码：改一个条件要改代码、重新测试、重新部署
- Skill配置：改YAML文件即可，不动代码，非技术人员也能看懂规则
- 而且Skill带了report_template，程序能自动生成客户看得懂的报告

**这就是你的核心能力**：不是写更多代码，而是设计更好的规则，让一段通用程序能处理几十个不同产品。

### 在行业里这叫什么

类似概念：LangChain的Agent配置、Anthropic的MCP、OpenAI的GPTs——用配置控制AI行为，而不是硬编码。我们在贷款领域的应用是首创，但方法论可以迁移到其他AI应用。

## 🎯 为什么选"贷款中介"这个领域？

### 你可能的疑问

**"为什么不做ChatGPT应用、AI绘画、智能客服这些热门赛道？"**

**我们的答案**：
我们故意选了一个**累、脏、小众**的市场。

### 为什么这样选？

**1. 避开大厂（活下来）**：
- 热门赛道：大厂重兵投入，创业公司活不过6个月
- 小众市场：大厂看不上，给创业公司生存空间
- 贷款中介：全国200万中介，但没有一家大厂做

**2. 有真实需求（能赚钱）**：
- 中小企业贷款：6000万家中小企业，贷款需求旺盛
- 中介痛点：存量客户流失、匹配效率低、改造方案靠经验
- 我们能解决：体质诊断、精准匹配、任务化改造

**3. 用AI重构传统行业（有想象力）**：
- 不是做"AI玩具"，而是"AI生产力工具"
- 目标：让中介从"关系生意"变成"数据+算法生意"
- 长期愿景：帮助中介升级为数据驱动的服务商，提升全行业效率

### 对你的价值

**技术价值**：
- 学的不是"做个聊天机器人"（烂大街）
- 而是"用AI重构一个行业"（稀缺）

**经验价值**：
- 简历上写"AI贷款诊断系统" > "AI客服"
- 面试时说"我用AI重构了贷款行业" > "我做了个ChatGPT插件"

**未来价值**：
- 如果项目做成，你是"用AI重构传统行业"的早期参与者
- 这个经验，比在大厂做螺丝钉值钱

### 如果贷款中介做不成呢？

**我们的答案**：
- 那就换个"累、脏、小众"的市场继续做
- 核心能力是"用AI重构传统行业"，不是"只会做贷款"

**对你的价值**：
