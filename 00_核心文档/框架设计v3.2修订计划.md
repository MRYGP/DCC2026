**合规侧**：

- **必须更"正规军"，而不是更像灰产工具**

**更新5.2节：验证问题清单 → 可验收指标**

```markdown
### 5.2 可验收指标（具体）⭐

**指标1：复活转化率**
- **定义**：导入的存量客户中，被系统触发并推进到有效沟通/进件的比例
- **目标**：>20%
- **验收方式**：系统记录进件数/导入总数

**指标2：筛选周期**
- **定义**：从接触客户到得出"能做/值得做/不建议"的平均时间
- **目标**：<3天
- **验收方式**：系统记录时间戳

**指标3：匹配准确率**
- **定义**：系统推荐的产品中，客户实际申请并通过的比例
- **目标**：>85%（预测）→ 实测
- **验收方式**：结果库回填

**指标4：中介续费率**
- **定义**：第一批中介在12个月后的续费比例
- **目标**：>70%
- **验收方式**：合同续签数据
```

### 修订5：新增2.0.5节"双语词典"⭐⭐⭐⭐

**插入位置**：2.0节"合规与可信"的最后一个小节

**修订内容**：
```markdown
#### 2.0.5 双语词典（对外中性，对内接地气）⭐

**原则**：
- 对外（企业端、会计端、合规）：使用中性词汇
- 对内（中介侧、团队内部）：使用接地气词汇
- 不同场景自动切换，避免误解

**词汇对照表**：

| 场景 | 对外表述 | 对内表述 |
|------|---------|---------|
| 企业端/会计端 | 提升路径 | 改造方案 |
| 企业端/会计端 | 进件准备清单 | 材料清单 |
| 合规/银行 | 融资体质评估 | 诊断 |
| 正式文档 | 历史未成交客户 | 沉睡客户/前女友 |
| 企业端/会计端 | 融资体质仪表盘 | 诊断系统 |
| 合规/银行 | 初步评估/建议 | 诊断结果 |

**为什么需要双语词典？**

对中介（接地气）：
- "改造"一听就懂，很直接
- "前女友"很形象，立即get到存量复活的意思
- "诊断"比"评估"更有专业感

对企业/会计（中性专业）：
- "改造"容易被误解为"我们要交付工程"
- "前女友"不够正式，可能显得不专业
- "提升路径"听起来更中性、更可接受

对合规/银行（严谨中立）：
- "诊断"要说成"初步评估"（避免医疗类比）
- "方案"要说成"建议"（避免承诺性表述）
- "改造"要说成"提升路径"（更中性）

**系统实现**：
- 系统内部有"语言模式"设置
- 根据不同租户类型（中介/会计/企业），自动切换措辞
- 文档生成时，根据目标受众选择不同词汇

**例子**：

```markdown
# 中介版报告
诊断结果：您的客户A001需要改造
缺口：开票不足（差20万）
推荐方案：开票提升术（改造方案002）

# 企业版报告
融资体质评估：初步评估结果
需要提升的方面：开票额度（当前30万，建议50万）
提升路径：6个月开票提升计划
```

**关键提醒**：
> 这不是"说一套做一套"，而是"同一件事，对不同人用不同语言说清楚"。
> 
> 就像医生对同行说"急性心肌梗死"，对患者说"心脏病发作"，本质是一回事。
```
