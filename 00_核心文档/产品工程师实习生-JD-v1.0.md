【产品工程师实习生】长沙坐班 | 3–4.5k/月 | 2 个月

我们在做什么

贷查查（DCC2026）：把银行贷款产品的准入规则写成 YAML 配置（我们叫 Skill），用程序跑匹配，自动判断「这个客户能不能贷、能贷多少、差在哪里」，让贷款中介告别靠记忆撞产品的时代。

核心不是写 CRUD，而是 理解规则 → 写配置 → 验证输出。

你会做的事

约 60% 时间：理解银行产品准入逻辑，设计 Skill 配置（YAML），验证匹配结果是否正确
约 40% 时间：写让 Skill 跑起来的代码（Python + React），写测试，写文档

8 周节奏：
Week 1–2：搭环境 + 理解贷款业务 + 写出第一个能跑通且有测试的 Skill
Week 3–4：把 3–5 个真实银行产品翻译成 Skill，写 matcher 跑通
Week 5–6：做前端（客户填表 + 报告展示），让中介能实际用
Week 7–8：集成测试 + 部署 + 复盘

2 个月结束，你能带走什么

全部可放 GitHub 公开展示，我们不限制：

3–5 个可运行银行产品 Skill（YAML，含 pytest）
1 个 Python matcher 函数（覆盖边界测试）
1 个前端报告展示页（React + Ant Design）
项目 README + 技术文档

写进简历时可以这样描述：「设计并实现 X 个银行产品 Skill 配置，覆盖准入条件/额度公式/边界测试，matcher 通过率 100%」。

岗位要求

必须会：
Python 基础（能写函数、类、会用框架）
Web 框架基础（Django / Flask / FastAPI 任选其一）
Git 基础（能提交代码、能解决冲突）
至少 1 个完整项目经验（课程项目也可以）

加分项：React 基础、用过 AI 工具（Cursor/Copilot）、写过技术文档

我们会培训：FastAPI、React 进阶、pytest、Skill 系统（Week 1–2）

工作安排

地点：长沙坐班为主。Day 1–3 远程试跑（不用辞职/请假）；之后每周至少 3 天长沙线下（硬要求）。无法来长沙的建议暂不投递。
时长：2 个月（8 周），通过验收后有转正机会
时间：全职每周 30–35 小时；更看重每天有连续 2–3 小时专注时段，不接受碎片化凑时长
薪资：3000–4500 元/月，面试按基础定档，通过 Week 2 验收后可微调

适合 / 不适合

更适合：大四/初级开发/转行程序员；能接受长沙坐班（每周至少 3 天）和连续 8 周；每天能有连续 2–3 小时专注。

不适合：更看重「写代码本身」、更看重技术栈新颖、更看重大厂高薪；不愿意承诺 8 周或每周 < 20h；「业务的事不管我只写代码」直接淘汰。
