# BOSS 招聘 JD 变更对照（v1.0）

**目标**：把 BOSS 上的职位描述与《招聘说明与申请指引-飞书版》《招聘文案 v2.0》对齐，避免候选人看到两套说法。

**使用方式**：按下面「应改项」修改 BOSS 职位描述；详细内容以飞书链接为准，BOSS 上可只保留简短标题 + 核心卖点 + 飞书链接。

---

## 一、标题与定位

| 原 BOSS JD | 应改为 / 建议 |
|------------|----------------|
| 【技术型AI应用开发实习生】长沙线下 \| 3-4.5k/月（优秀候选人可议） | **【产品工程师实习生】长沙坐班 \| 3–4.5k/月**（或直接写「详见下方链接」） |
| 强调「技术型」「AI应用开发」 | 当前定位是 **产品工程师**：60% 理解规则 + 写 Skill 配置，40% 写代码跑通 + 测试；核心是「把银行规则写成 YAML 配置」，不是泛泛的「AI 应用开发」。 |

## 二、项目方向与「你会学到」

| 原 BOSS JD | 应改为 / 建议 |
|------------|----------------|
| 参与"贷查查"AI产品开发，从0到1搭建完整应用 | 参与贷查查（DCC2026）**规则引擎**产品开发：把银行产品准入写成 **Skill（YAML 配置）**，用程序跑匹配并生成报告。从 0 到 1 跑通完整闭环。 |
| 第1个月：问卷+计算器+报告生成；第2-3个月：迭代优化 | 当前口径是 **2 个月（8 周）**：Week 1–2 环境+第一个 Skill；Week 3–4 多个银行产品 Skill+跑通；Week 5–6 前端（客户填表+报告）；Week 7–8 集成测试+部署。没有「问卷+计算器」的泛称，而是 **客户填表 + 匹配引擎 + 报告**，且核心是 **Skill 配置**。 |
| 深度拆解 Perplexity、Cursor 等，提炼方法论 | 可保留「会花 20–30% 时间探索其他 AI 方向（如 AI 拆解仓库）」，但不要写成「第1个月做 MVP、第2–3月拆解」——当前主文档是 **8 周任务清单**，拆解是副线。 |

## 三、核心价值与工作职责

| 原 BOSS JD | 应改为 / 建议 |
|------------|----------------|
| 核心开发 80% + AI 工具探索 20% | **60% 理解业务 + 设计 Skill 配置 + 验证输出；40% 写让 Skill 跑起来的代码。** 不写「AI 工具探索 20%」占比，避免让人以为主活是「调研 Bolt.new、v0、Dify」。 |
| 问卷模块、计算器引擎、自动生成报告 | **客户信息录入、Skill 匹配引擎、可解释报告生成**；强调「准入条件/额度公式/边界测试」都在 **YAML Skill** 里，不是硬编码。 |
| 周一-周四开发，周五部署 | 可保留「典型节奏」类描述，但不要与《Week1–8 任务清单》冲突；当前文档未强调周几部署。 |

## 四、技术栈与招聘人数

| 原 BOSS JD | 应改为 / 建议 |
|------------|----------------|
| 前端 1 名 + 后端 1 名；前端 React/Vue，后端 Node.js/Python | **主方向可选：后端 / 前端 / 全栈。** 技术栈固定：**后端 Python（FastAPI）**，**前端 React + Ant Design**。不招「Node 后端」或「Vue 前端」为主的技术栈；若写「招 2 名」，可写「2 名实习生，主方向可前端/后端/全栈互补」。 |
| 后端：Node.js 或 Python，有部署经验 | **后端必须 Python**（Django/Flask/FastAPI 其一）；部署可后续培训。与《术语与决策 SSOT》一致。 |

## 五、工作安排

| 原 BOSS JD | 应改为 / 建议 |
|------------|----------------|
| 时长：3–6 个月 | **实习期 2 个月（8 周）**；转正后另谈。当前主文档无「3–6 个月」说法。 |
| 每周 25–30 小时 | **全职**：Week 1–2 约 25–30h/周，Week 3–8 约 30–35h/周。**非全职**：15–20h → 20–25h/周。并注明：**更看重每天连续 2–3 小时专注**，不接受碎片化凑时长。 |
| 地点：长沙线下办公 | **长沙坐班为主**：Day 1–3 远程试跑，之后 **每周至少 3 天长沙线下（硬要求）**；毕设/答辩周可协商豁免。无法来长沙的建议等开放纯远程再申请。 |

## 八、适合 / 不适合

| 原 BOSS JD | 应改为 / 建议 |
|------------|----------------|
| 可能不太适合：只想混实习证明、完全不愿学新工具、每周只能很少时间 | 与飞书版一致：**不适合**——更看重「写代码本身」、更看重技术栈新颖、更看重大厂高薪；**且**明确：不愿意承诺 8 周或每周 &lt; 20h 一般不进入小作业；「业务的事不管我只写代码」直接淘汰。 |
| 更适合：大三/大四、React 或部署基础、对 AI 感兴趣、25–30h | 保持「适合人群」：大四/初级开发/转行程序员；**必须**能接受长沙坐班（每周至少 3 天）和连续 8 周；每天能有连续 2–3 小时专注。 |

## 九、投递方式与第一句话

| 原 BOSS JD | 应改为 / 建议 |
|------------|----------------|

## 十、建议的 BOSS 职位描述（简短版）

以下可作为 BOSS 职位描述正文，其余细节一律指向飞书链接。

---

**【产品工程师实习生】长沙坐班 | 3–4.5k/月 | 2 个月**

我们做贷查查（DCC2026）：把银行产品准入规则写成 YAML 配置（Skill），用程序跑匹配并生成报告。核心不是写 CRUD，而是**理解规则 + 写配置 + 验证输出**。

- **你会做**：60% 理解业务、设计 Skill、验证结果；40% 写代码让 Skill 跑起来（Python + React）。
- **节奏**：2 个月，Week 1–2 环境+第一个 Skill，Week 3–6 多产品 Skill+前端报告，Week 7–8 集成与部署。
- **要求**：Python + Web 基础 + Git + 1 个完整项目；能长沙坐班（Day 1–3 远程试跑后每周至少 3 天线下），连续 8 周，每天有连续 2–3 小时专注时间。
- **申请**：请先看完整说明（下方链接），按文档内流程发邮件。3 个工作日内回复。
