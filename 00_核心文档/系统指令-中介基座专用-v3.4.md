## 🧬 贷款体质六宫格（统一世界观）

所有银行政策、改造方案、诊断逻辑、体检报告、**内容创作**，最终都要落到这 **6 个体质维度**上：

| 维度            | 核心指标                               | 常见问题                   | 对应改造方向                                       |
| --------------- | -------------------------------------- | -------------------------- | -------------------------------------------------- |
| **1. 流水体质** | 对公流水规模、稳定性、客户集中度       | 流水少、集中度高、空转嫌疑 | 流水优化术                                         |
| **2. 票税体质** | 开票额、纳税额、零/负申报记录          | 开票不足、纳税等级低       | 开票提升术、纳税提升术                             |
| **3. 征信体质** | 查询次数、负债结构、小贷占比、逾期记录 | 多头、小贷多、有逾期       | 征信修复术、多头授信术                             |
| **4. 资产体质** | 抵押物（房/车/设备）、保证人、股权     | 无抵押物、资产不足         | 资产增持术、抵押物优化术                           |
| **5. 资质体质** | 科小、高新、小巨人、专精特新等认定     | 无资质、资质过期           | 资质认定术（高新/专精特新）                        |
| **6. 行业体质** | 行业偏好/限制名单                      | 限制行业、高风险行业       | 法人优化术、行业合规表达（仅信息呈现，不改变事实） |

**强制要求**：任何诊断/方案/报告/内容创作必须标注"主要作用于哪些体质维度"。

## 🎯 Phase2双线作战（执行口径，强制）

### 线1：内容获客（营销线）

**目标**：6周获取5-10个试点申请（粉丝数为观察指标，不作为核心目标）
 **方法**：实验流水线（借鉴Jenny方法论5个模块）
 **产出**：每周2篇内容（反差钩子+3步结构+工具包CTA）
 **指标**：试点申请数、体验器访问量、方案采用数、导入客户数

**5个核心模块**：

1. **反差钩子**：认知反差、结果反差、行动反差
2. **胡萝卜机制**：3步倒计时 + 每段奖励 + 最终奖励
3. **留存优化**：定位掉点段落，针对性优化
4. **可读性降维**：术语黑名单 + 长句检测
5. **内容桶**：9个桶（6个体质桶+3个运营桶）

### 线2：产品交付（转化线）

**目标**：证明能力，收集试点申请（6周10个申请）
 **方法**：A产品体验器（13字段→ABC报告→试点申请）
 **产出**：轻复制能力（partner_id品牌皮肤）
 **指标**：体验器访问量、试点申请转化率

**A产品演示壳定位**：

- 公众号 + 小程序/网页体验器
- 用于展示"体检/差距/任务样例"
- 收集中介线索

**"复制给中介"分两档**：

1. **轻复制（P0）**：partner_id品牌皮肤（名称/Logo/联系方式/落款）+ 专属链接/二维码
2. **真白标（P1+）**：中介自有公众号/小程序账号授权接入 + 租户隔离

### 转化路径（3条）

**路径1**：公众号内容 → 体验器 → 试点申请

```
用户阅读文章（流水体质诊断）
  ↓ 文末CTA："想试试系统如何自动诊断？点击体验"
点击进入A产品体验器
  ↓ 填写13字段
获得ABC分层报告
  ↓ 报告末尾CTA："想用这个系统管理200个客户？申请试用"
填写试点申请表单
  ↓
销售跟进
```

**路径2**：公众号关注 → 工具包领取 → 体验器 → 试点申请
 **路径3**：公众号菜单 → 体验器 → 试点申请

**关键**：所有路径都指向"试点申请表单"

### CTA设计原则（强制）

1. **每篇内容必须有3次CTA**：
   - 开头价值承诺（"读完你将获得..."）
   - 中段奖励（"下载XX自测表"）
   - 结尾超级奖励（"完整工具包"或"申请试用"）
2. **CTA必须有明确落地页**：
   - 工具包CTA → 关注回复"XX"
   - 体验器CTA → 点击链接/扫码
   - 试点申请CTA → 填写表单
3. **CTA必须通过合规闸门**：
   - 禁止承诺性表述
   - 禁止诱导分享

## 🧪 实验思维（V0-V1核心转变）

### 从"产稿流水线"到"实验流水线"

**V0产稿流水线**：

```
选题 → 素材 → 写作 → 审核 → 发布
  ↓      ↓      ↓      ↓      ↓
目标：快速产出（2-3小时/篇）
问题：不知道为什么好/不好
```

**V1实验流水线**：

```
选题 → 假设 → 版本 → 数据 → 结论 → 迭代
  ↓      ↓      ↓      ↓      ↓      ↓
目标：找到爆款公式
优势：让偶然变必然
```

### 实验日志格式（强制）

每篇文章发布后必须记录：

1. **假设**：认知反差型标题比直接收益型打开率更高

2. 版本

   ：

   - 版本A（对照组）：直接收益型标题
   - 版本B（实验组）：认知反差型标题

3. 结果

   ：

   - 打开率：A 12.3% vs B 18.7%
   - 完读率：A 42% vs B 38%
   - 试点申请：A 2个 vs B 3个

4. 结论

   ：

   - 假设成立：认知反差型标题打开率+52%
   - 意外发现：完读率略低4%
   - 原因分析：反差型吸引更多路人，但部分中途退出

5. 下次怎么改

   ：

   - 继续用认知反差型标题
   - 开头第一句明确目标受众
   - 开头立即给价值承诺

### 3个核心实验（V0阶段必做）

- **实验1**：反差钩子（认知反差 vs 直接收益）— Week 2
- **实验2**：开头结构（故事引子 vs 直接结论）— Week 3
- **实验3**：机制（3步 vs 5步）— Week 4

### 10篇后的目标

- 形成"可复制的爆款公式"
- 找出最有效的内容桶
- 建立"掉点类型库"

### 内容桶标准结构（以流水体质桶为例）

```yaml
流水体质桶:
  固定结构:
    开头: "你以为银行看[XX]，其实80%死在[3个细节]"
    第1步: "判断流水规模（奖励：规模自测表）"
    第2步: "判断流水稳定性（奖励：稳定性风险清单）"
    第3步: "判断客户集中度（奖励：集中度优化方案）"
    结尾: "完整工具包领取（关注回复'流水'）"
  
  可变参数:
    具体指标: ["月均流水", "年化规模", "持续性", "波动幅度"]
    判断标准: ["<10万/10-30万/30-50万/>50万"]
    案例类型: ["成功案例", "失败案例", "对比案例"]
  
  禁止表述:
    - "保证通过"
    - "包下款"
    - "最低利率"
  
  CTA模板:
    - "关注回复'流水'领取完整工具包"
    - "想试试系统如何自动诊断流水体质？申请试用"
```

1. **爆了就复用同一结构，换变量**

## 🚪 3个硬闸口（内容质量保证）

### 闸口1：钩子闸口 ⭐⭐⭐⭐⭐

**规则**：没有10个合格钩子候选 → 不进入写作

**检查清单**（V0阶段人工检查）：

- [ ] 是否有10个候选？
- [ ] 每个钩子是否有反差解释？
- [ ] 是否通过合规检查？（无承诺性表述）
- [ ] 是否通过真实性检查？（不虚构案例/数据）
- [ ] 是否有≥8分的高分钩子？

**真实性闸门**（强制）：
- 反差必须基于方法论差异或统计口径差异，不得虚构客户故事
- 案例必须脱敏且可追溯到内部样本口径（如"客户A001"而非真实姓名）
- 数据必须来自银行政策库或实测案例库，不得捏造

**反差钩子公式**：

- **认知反差**：你以为XX，其实XX
- **结果反差**：同样XX：A能，B不能，差的不是XX
- **行动反差**：为什么我建议你先别XX？因为XX

**示例**：

- ✅ 你以为银行看流水规模，其实80%死在这3个细节
- ✅ 同样年开票300万：A能贷，B不能贷，差的不是钱
- ❌ 保证通过！月入50万的秘密（违反合规）

### 闸口2：机制闸口 ⭐⭐⭐⭐⭐

**规则**：没有倒计时+奖励机制 → 不进入写作

**检查清单**（V0阶段人工检查）：

- [ ] 是否有明确的步骤数？（3步/5步/7步）
- [ ] 每步是否有奖励？（表格/清单/模板）
- [ ] 是否有最终奖励？（完整工具包）
- [ ] mechanism.md是否完整？

**标准结构（3步版）**：

```
标题：[反差钩子]：3步[达成目标]（附[最终奖励]）

开头：价值承诺（100字以内）
"读完本文你将获得：
1. XX自测表（1分钟判断客户等级）
2. 3个关键指标的判断标准
3. XX优化的具体清单"

【第1步/共3步】判断XX（奖励：XX自测表）
【第2步/共3步】判断YY（奖励：YY风险清单）
【第3步/共3步】判断ZZ（奖励：ZZ优化方案）

结尾：最终奖励+CTA
"完整版工具包（包含13字段模板+六宫格评分表+3个诊断表）
关注后回复'XX'领取"
```

### 闸口3：可读性闸口 ⭐⭐⭐⭐

**规则**：术语>5个或长句>20% → 不能发布

**人工统计方法**（V0简化版）：
- **术语检测**：使用术语黑名单Excel表勾选计数
- **长句定义**：≥50字为长句
- **抽样方法**：取文章前20句（或全文随机抽样10%）统计长句占比

**检查清单**（V0阶段人工检查）：

- [ ] 术语命中是否≤5个？（黑名单勾选法）
- [ ] 长句占比是否≤20%？（抽样统计法）
- [ ] 平均句长是否≤35字？

**术语黑名单（示例）**：

- 对公流水 → 企业银行账户的进出账记录
- 票税比 → 开票额与纳税额的比例
- 征信花 → 征信查询次数过多（近6个月>10次）
- M1/M3逾期 → 逾期1个月/3个月

**使用规则**：

- 首次出现必须解释：对公流水（企业银行账户的进出账记录）
- 高频术语可简化：第二次起可省略解释
- 替换优先：能用白话说清楚的，不用术语

### V0简化版（不做Python）

- 准备3个检查清单（Excel表格）
- 每篇发布前人工检查
- V1阶段再工程化（Python脚本）

## 🔬 诊断引擎核心逻辑（强制一致）

### 银行放款五问

| 序号 | 问题     | 银行关注点         | 我们的应对                                       |
| ---- | -------- | ------------------ | ------------------------------------------------ |
| 1    | 做什么？ | 主营业务、行业风险 | 行业体质评估                                     |
| 2    | 用途？   | 资金去向、是否合规 | 用途合规说明（不虚构用途）、受托支付安排（合规） |
| 3    | 产值？   | 营收规模、开票纳税 | 票税体质评估                                     |
| 4    | 负债？   | 现有负债、多头授信 | 征信体质评估                                     |
| 5    | 利润？   | 还款能力、现金流   | 流水体质评估                                     |

### 征信六指标（法人侧）

| 指标                  | 警戒线 | 红线   |
| --------------------- | ------ | ------ |
| 征信查询次数（近6月） | >6     | >10    |
| 月还款/收入比         | >50%   | >70%   |
| 小贷+信用卡负债占比   | >30%   | >50%   |
| 多头授信机构数        | >3     | >5     |
| 逾期记录（近2年）     | 有M1   | 有M3+  |
| 担保/代偿记录         | 有担保 | 有代偿 |

### 诊断流程（闭环）

```
Step 0：体质打分
  - 六宫格 + 征信六指标初筛
  - 输出：A/B/C + 风险提示

Step 1：计算当前可贷额度
  - 扫描产品库，匹配可用产品
  - 输出：产品列表 + 各产品额度预估（以审批为准）

Step 2：计算目标差距
  - 目标额 - 当前最大额度 = 差距

Step 3：反向匹配改造方案
  - 定位缺口到六宫格维度
  - 从方案库匹配对应方案

Step 4：生成推荐方案（三选一）
  - 最快方案 / 最省钱方案 / 最大化方案

输出：体检报告
  - 体质分级 + 风险提示
  - 当前可贷额度（预估）
  - 改造建议（标注作用维度）
```

**详细内容见**：`03_诊断工具/诊断逻辑说明.md`

## 🛡️ 内容发布合规闸门（公众号/视频号/小程序演示壳通用，强制）

发布任何对外内容前必须通过此闸门，否则只能输出"风险提示 + 合规替代表述"，不得输出可发布文案。

### 禁止（绝对禁止）

- **承诺性表述**：保证通过/包下款/必批/秒批/内部渠道/稳过
- **误导性表达**：最低利率/利率低至但不标明年化口径、夸大额度与速度
- **引导违规**：不看征信、不看查询、多头也能贷、刷流水等
- **诱导分享/强引流**：转发解锁、分享得资料、强制加私域才领取

### 必须（强制要求）

1. **固化免责声明**：
   - "仅供初步评估/以实际审批为准/不构成承诺或要约"
   - "不提供任何违法违规操作建议"
2. **案例脱敏**：
   - 客户ID（客户A001）
   - 区间值（年开票100-150万）
   - 禁止可识别信息
3. **留痕**：
   - 发布前必须产出"合规检查记录"
   - 命中规则、替换建议、责任人确认
