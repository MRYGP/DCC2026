# 【六宫格体检课】模板

## 模板基本信息
- **模板类型**：六宫格体检课
- **目标读者**：贷款中介（1-5人小团队）
- **阅读时间**：5-8分钟
- **核心价值**：提供六个维度的诊断方法论
- **模板版本**：v1.0 (2026-02-03)
- **V0对齐**：服务V0试点招募，辅助产出方法论标准化内容

## 标题公式

### V0合规标题格式
```
《[维度]体质诊断：如何判断客户[维度]是否健康？》
```
**示例**：《流水体质诊断：如何判断客户流水是否健康？》

### 选题方向（V0阶段）
1. 《流水体质诊断：对公流水的6个关键指标》
2. 《票税体质评估：开票额与纳税额的匹配度》
3. 《征信体质检查：法人征信的6个红线》
4. 《资产体质盘点：抵押物价值评估方法》
5. 《资质体质审核：高新/专精特新的真实价值》
6. 《行业体质判断：哪些行业容易被银行拒绝？》

### V0禁语清单（必须规避）
以下表述在标题和正文中都**绝对禁止**：
- ❌ 保证/包过/必下款/秒批/100%通过
- ❌ 最低利率/最高额度/最优方案
- ❌ 具体贷款额度数值（如"贷到200万"）
- ❌ 确定时间承诺（如"90天拿到贷款"）
- ❌ 内部渠道/特殊关系/独家方案
- ❌ 哪家银行最低/哪家最快/XX银行最好

### V0合规表述替代
将结果承诺改为过程与边界描述：
- ❌ "保证通过" → ✅ "达到银行准入标准的具体步骤"
- ❌ "最低利率" → ✅ "符合当前市场水平的利率区间"
- ❌ "分析了500个案例" → ✅ "从我们观察到的案例中总结"

## 文章结构（共6部分，约1500字）

### 第一部分：维度定义（200-300字）
**目的**：明确诊断对象，建立专业认知

**合规要点**：
- 使用中性描述，不承诺结果
- 引用银行官方标准，不添加主观判断
- 明确适用范围和边界条件

**内容要点**：
```
【维度说明】
- 维度名称：[流水/票税/征信/资产/资质/行业]
- 银行视角：为什么银行重视这个维度？
- 中介痛点：中介在这个维度常遇到的问题

【重要性分析】
- 影响哪些贷款产品？
- 在审批中的权重占比？
- 常见误区和认知偏差
```

**写作提示**：
- 用银行官方文件/标准作为依据
- 突出"可量化、可诊断"的特点
- 设置悬念（如何科学诊断？）

### 第二部分：关键指标（300-400字）
**目的**：提供具体可操作的诊断标准

**内容要点**：
```
【核心指标清单】
1. 指标1：[具体指标名称]
   - 警戒线：[银行通常的要求]
   - 数据来源：[如何获取这个数据]
   - 验证方法：[如何验证准确性]

2. 指标2：[具体指标名称]
   - 警戒线：[银行通常的要求]
   - 数据来源：[如何获取这个数据]
   - 验证方法：[如何验证准确性]

【指标关系图】
- 指标之间的相互影响
- 哪个指标最关键？
- 诊断的优先级顺序
```

**写作提示**：
- 提供具体的数值区间（非精确值）
- 用表格对比更清晰
- 提供Excel模板下载链接（如果适用）

### 第三部分：诊断方法（400-500字）
**目的**：提供可执行的诊断步骤

**内容要点**：
```
【诊断步骤】
Step 1：[第一步做什么]
  - 工具需求：[需要什么工具]
  - 操作说明：[具体操作方法]
  - 预计用时：[大概需要多长时间]

Step 2：[第二步做什么]
  - 工具需求：[需要什么工具]
  - 操作说明：[具体操作方法]
  - 预计用时：[大概需要多长时间]

【风险等级划分】
- 绿色（健康）：[具体标准]
- 黄色（预警）：[具体标准]
- 红色（危险）：[具体标准]
```

**写作提示**：
- 分步骤说明，让读者能跟着做
- 强调"可执行性"和"可验证性"
- 提供工具/模板下载链接

### 第四部分：改善方向（300-400字）
**目的**：提供问题解决思路

**内容要点**：
```
【改善路径】
- 短期改善：[3个月内可做的动作]
- 中期改善：[3-6个月可做的动作]
- 长期改善：[6个月以上可做的动作]

【对应改造方案】
- 轻度问题 → [对应哪个改造方案]
- 中度问题 → [对应哪个改造方案]
- 重度问题 → [对应哪个改造方案]

【ROI分析】
- 时间投入：[需要多长时间]
- 成本预算：[大概需要多少钱]
- 预期收益：[可能提升多少额度]
```

**写作提示**：
- 提供具体的时间规划和成本估算
- 强调"投入产出比"
- 链接到任务化改造方案栏目

### 第五部分：案例示范（200-300字）
**目的**：增加可信度和实操感

**内容要点**：
```
【案例背景（脱敏版）】
- 客户代号：[客户A001]
- 原始状态：[维度健康状况]
- 诊断过程：[具体诊断步骤]
- 改善结果：[改善后的状态]

【关键洞察】
- 最容易被忽视的点是什么？
- 诊断中的常见错误有哪些？
- 成功的关键因素是什么？
```

**写作提示**：
- 案例必须脱敏处理
- 使用区间值，避免精确值
- 突出"典型性"（很多中介都遇到过）

### 第六部分：工具下载（100-200字）
**目的**：提供可立即使用的工具，引导试点申请

**内容要点**：
```
【工具清单】
1. 《[维度]诊断清单》Excel模板
   - 包含内容：[具体包含哪些内容]
   - 使用说明：[如何使用]
   - 下载方式：[如何获取]

2. 《客户[维度]评估表》
   - 包含内容：[具体包含哪些内容]
   - 使用说明：[如何使用]
   - 下载方式：[如何获取]

【V0试点申请】
- 想体验系统如何帮你诊断客户？
- 点击申请V0试点资格（限3-5个名额）
- 或关注后回复"试点"获取申请表单
```

**写作提示**：
- 提供真实可用的工具
- 明确下载方式和操作说明
- 引导到V0试点申请

## V0审核检查清单

## 模板使用说明（V0简化版）

### V0使用流程
1. **选择维度**：从6个维度中选择1个
2. **收集素材**：整理银行标准、案例素材、工具模板
3. **填充模板**：按照6部分结构填充
4. **合规检查**：通过7项强制检查
5. **人工审核**：通过人工审核清单
6. **发布引导**：引导到试点申请

- **人力**：1人兼职（5-10小时/周）
