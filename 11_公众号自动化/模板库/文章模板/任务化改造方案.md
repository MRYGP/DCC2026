# 【任务化改造方案】模板

## 模板基本信息
- **模板类型**：任务化改造方案
- **目标读者**：贷款中介（1-5人小团队）
- **阅读时间**：5-8分钟
- **核心价值**：提供具体可执行的任务清单
- **模板版本**：v1.0 (2026-02-03)
- **V0对齐**：服务V0试点招募，展示产品任务化能力

## 标题公式

### V0合规标题格式
```
《[问题]改造方案：具体任务清单+证据要求+ROI分析》
```
**示例**：《开票不足改造方案：3个月提升50万的任务清单》

### 选题方向（V0阶段）
1. 《开票不足改造：3个月从30万到80万的任务包》
2. 《征信多头改造：90天降低负债率的执行清单》
3. 《流水优化改造：如何提升流水稳定性？》
4. 《高新认定改造：从申请到获批的完整任务链》

### V0禁语清单（必须规避）
以下表述在标题和正文中都**绝对禁止**：
- ❌ 保证/包过/必下款/秒批/100%通过
- ❌ 最低利率/最高额度/最优方案
- ❌ 具体贷款额度数值（如"贷到200万"）
- ❌ 确定时间承诺（如"90天拿到贷款"）
- ❌ 内部渠道/特殊关系/独家方案
- ❌ 哪家银行最低/哪家最快/XX银行最好

### V0合规表述替代
将结果承诺改为过程与边界描述：
- ❌ "90天贷到200万" → ✅ "90天把'票税体质'补到可申条件：任务清单+证据要求"
- ❌ "保证通过" → ✅ "达到银行准入标准的具体步骤"
- ❌ "分析了500个案例" → ✅ "从我们观察到的案例中总结"

## 文章结构（共7部分，约1500字）

### 第一部分：问题诊断（200-300字）
**目的**：明确问题所在，建立诊断权威

**合规要点**：
- 使用中性描述，不承诺结果
- 引用银行官方标准，不添加主观判断
- 明确适用范围和边界条件

**内容要点**：
```
【客户基本情况（脱敏）】
- 客户代号：[客户A001]
- 成立时间：[区间，如3-5年]
- 注册资本：[区间，如300-500万元]
- 主营行业：[大类描述，如科技/贸易/制造]

【具体差距分析】
- 银行要求：[具体产品准入标准]
- 客户现状：[当前状况描述]
- 关键差距：[差距量化描述]
- 影响产品：[影响哪些贷款产品]
```

**写作提示**：
- 用银行官方文件/标准作为依据
- 差距描述使用区间值
- 突出"可量化、可诊断"的特点

### 第二部分：改造目标（100-200字）
**目的**：明确改造方向和期望结果

**内容要点**：
```
【改造目标（区间描述）】
- 目标额度：[期望贷到的额度区间]
- 改造周期：[预计需要的时间区间]
- 成本预算：[预计投入的成本区间]

【关键成功指标】
- 必须完成的里程碑
- 银行关注的核心数据
- 改造完成的标准
```

**写作提示**：
- 目标使用区间值，避免精确值
- 强调"可衡量、可实现"
- 提供时间规划参考

### 第三部分：任务清单（500-600字）⭐核心部分
**目的**：提供具体可执行的任务步骤

**内容要点**：
```
【任务包分解】
```
任务1：[具体任务名称]
├─ 负责人：[谁来做]
├─ 截止时间：[多久完成]
├─ 证据要求：[需要什么材料]
└─ 完成标准：[怎样算完成]

任务2：[具体任务名称]
├─ 负责人：[谁来做]
├─ 截止时间：[多久完成]
├─ 证据要求：[需要什么材料]
└─ 完成标准：[怎样算完成]
```

【任务依赖关系】
- 哪些任务必须按顺序执行？
- 哪些任务可以并行执行？
- 关键路径是什么？
```

**写作提示**：
- 任务描述具体可执行
- 负责人明确，截止时间合理
- 证据要求清晰具体
- 完成标准可验证

### 第四部分：证据清单（200-300字）
**目的**：提供具体证据要求和材料标准

**内容要点**：
```
【证据材料清单】
1. 证据1：[具体证据名称]
   - 收集方法：[如何获取]
   - 格式要求：[需要什么格式]
   - 提交时间：[什么时候提交]
   - 核验标准：[怎样算合格]

2. 证据2：[具体证据名称]
   - 收集方法：[如何获取]
   - 格式要求：[需要什么格式]
   - 提交时间：[什么时候提交]
   - 核验标准：[怎样算合格]

【证据归档要求】
- 如何整理证据？
- 如何提交给银行？
- 注意事项有哪些？
```

**写作提示**：
- 证据要求具体可行
- 提供格式模板下载
- 强调证据真实性和完整性

### 第五部分：复评机制（200-300字）
**目的**：提供持续监控和改进机制

**内容要点**：
```
【复评时间节点】
- 第1个月复评：[具体检查内容]
- 第3个月复评：[具体检查内容]
- 第6个月复评：[具体检查内容]

【复评标准】
- 进度检查标准：[怎样算按计划进行]
- 效果评估标准：[怎样算效果良好]
- 调整建议标准：[什么时候需要调整]

【系统自动提醒】
- 提醒设置：[如何设置提醒]
- 提醒内容：[提醒包含什么信息]
- 调整建议：[如何根据提醒调整]
```

**写作提示**：
- 复评标准具体可操作
- 提供复评时间规划参考
- 强调"持续改进"的重要性

### 第六部分：ROI分析（200-300字）
**目的**：提供投入产出分析，增强说服力

**内容要点**：
```
【投入成本分析】
- 时间投入：[需要多长时间]
- 资金投入：[需要多少钱]
- 人力投入：[需要多少人]
- 机会成本：[可能放弃的其他机会]

【预期收益分析】
- 贷款额度提升：[可能提升多少额度]
- 中介服务费：[可能增加多少收入]
- 客户关系深化：[可能带来的长期价值]
- 品牌影响提升：[可能增强的市场地位]

【回本周期估算】
- 保守估计：[多长时间]
- 乐观估计：[多长时间]
- 风险提示：[可能影响回本的因素]
```

**写作提示**：
- 分析基于区间值，避免精确值
- 提供多种情景分析
- 强调"理性决策"的重要性

### 第七部分：风险提示（100-200字）
**目的**：提示潜在风险，提供规避建议

**内容要点**：
```
【主要风险提示】
1. 政策风险：[银行政策可能变化]
2. 执行风险：[方案执行效果因人而异]
3. 时间风险：[实际时间可能超过预期]
4. 成本风险：[实际成本可能超过预算]

【风险规避建议】
- 如何监测政策变化？
- 如何确保执行效果？
- 如何控制时间和成本？
- 如何应对意外情况？

【免责声明（必加）】
⚠️ 本文内容仅供参考，不构成任何承诺或保证：
1. 所有贷款申请以银行实际审批为准
2. 改造方案效果因人而异，不保证必然通过
3. 实际时间、成本可能与估算不同
4. 如需具体建议，请联系专业顾问
```

**写作提示**：
- 风险提示具体清晰
- 规避建议切实可行
- 免责声明完整规范

## V0审核检查清单

### V0合规性检查（7项强制检查）
- [ ] 1. 承诺性表述检查：无"保证通过"、"包下款"等表述
- [ ] 2. 产品排名检查：无"哪家最低"、"哪家最快"等表述
- [ ] 3. 数据真实性检查：无"平均激活率21.3%"等不可验证数据
- [ ] 4. 诱导分享检查：无"转发海报解锁"等诱导分享机制
- [ ] 5. 免责声明检查：包含完整V0免责声明模板
- [ ] 6. 案例脱敏检查：案例已脱敏处理，无真实信息
- [ ] 7. CTA合规检查：行动号召合规，无"立即下款"等表述

### 事实准确性检查
- [ ] 所有数据都有来源依据

- [ ] ROI分析基于合理假设

### 专业性检查

## 模板使用说明（V0简化版）

### V0使用流程
1. **选择问题**：从常见问题中选择1个
2. **收集素材**：整理银行标准、案例素材、任务模板
3. **填充模板**：按照7部分结构填充
4. **合规检查**：通过7项强制检查
5. **人工审核**：通过人工审核清单
6. **发布引导**：引导到试点申请

### V0资源约束
- **人力**：1人兼职（5-10小时/周）
- **时间**：目标2-3小时/篇（含审核）
- **频率**：每周1篇（任务化改造方案）
- **目标**：服务V0试点招募，展示产品任务化能力
